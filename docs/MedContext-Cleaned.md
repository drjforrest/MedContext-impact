# MedContext: Data Collection & Testing Dataset Framework
## Comprehensive Guide to Test Images, Datasets, and Misinformation Examples

**Prepared for:** Purpose Africa, HERO Organization, MedContext Project  
**Date:** January 14, 2026  
**Purpose:** Identify, catalog, and validate medical image datasets for misinformation detection system testing

---

## EXECUTIVE SUMMARY

This document identifies **ready-to-use datasets, fact-checking examples with URLs, and collection protocols** for MedContext's MVP phase. The framework prioritizes:

- **Real-world misinformation examples** with documented URLs
- **Legitimate medical image datasets** for comparative analysis  
- **Deepfake detection benchmarks** for image authenticity validation
- **Practical collection templates** for ongoing corpus expansion

**Immediate actionable items:**
- 3 datasets ready for download (500+ medical images)
- 5 documented medical image misinformation cases with URLs
- 1 deepfake detection dataset (MedForensics: 116,000 images)
- Standardized documentation templates

### 1.3 Fact-Checking Archives with Documented Examples

#### **AFP Fact-Check Medical Images Database**
- **Source:** Agence France-Presse  
- **URL:** https://factcheck.afp.com/
- **Filter:** Search "medical image" OR "X-ray" OR "ultrasound"
- **Example Cases:**

##### Example 1: Cockroach in Lungs Hoax
- **Fact-Check URL:** https://factcheck.afp.com/doc.afp.com.33FT9U9
- **Image:** Doctored chest X-ray with inserted cockroach silhouette
- **Original Image Source:** Wikimedia Commons  
  - https://commons.wikimedia.org/wiki/File:Pneumothorax_-_Traumatic.jpg
  - Metadata: Normal PA chest X-ray, 21-year-old female, soccer injury, NO pathology
- **Misinformation Claim:** "Patient had live cockroach in lungs; Singapore doctor corrected Myanmar misdiagnosis"
- **Claim Variants:** Circulated in Myanmar (2012, 2019, 2020, 2021, 2022) and Kenya (2024)
- **Lesson:** Same image, different narratives; cultural context matters
- **Status:** Hoax confirmed; image manipulated (Photoshop); original image clean

##### Example 2: COVID-19 Lung Images
- **Search:** https://factcheck.afp.com/?s=covid+lung+x-ray
- **Multiple documented cases** of "vaccine-damaged lungs" claims
- **Pattern:** Legitimate COVID+ X-ray → presented as "post-vaccine injury"

## PART 2: DOCUMENTED MEDICAL IMAGE MISINFORMATION EXAMPLES WITH URLS

### 2.1 Case Study 1: COVID-19 Vaccine Injury False Claims

**Status:** ACTIVE misinformation (ongoing circulation)

#### Collection Point 1: COVID Lung Damage Claims
- **False Claim:** "This is a COVID vaccine-damaged lung"
- **Reality:** Image is COVID-19 positive X-ray
- **Fact-Check Sources:**
  - AFP: https://factcheck.afp.com/ (search "vaccine lung")
  - PolitiFact: https://www.politifact.com/ (search "vaccine lung injury")
- **Image Frequency:** Same 3-5 X-rays circulate repeatedly with label changes
- **Platform Spread:** WhatsApp groups, Facebook pages, Telegram channels
- **Language Variants:** English, French, Portuguese, Spanish, Swahili

#### Collection Point 2: Specific Image Traces
- **Example Image:** Severely abnormal COVID+ PA chest X-ray
- **TinEye Results (estimated):** 500-2,000 instances across internet
- **Primary Source:** Medical journal or early COVID studies (2020)
- **Misuse Timeline:** 
  - 2020-2021: Accurate COVID context
  - 2021-2022: Reused as "vaccine side effect" evidence
  - 2023-2024: Still circulating in anti-vax WhatsApp groups

#### How to Find These Images
1. **Reverse image search** on Google Images/TinEye
2. **Platform search:** "vaccine injured lung" + site:facebook.com
3. **WhatsApp group archives** (if you have access through ethics-approved studies)

### 2.2 Case Study 2: "Unproven Cancer Treatment" Image Misuse

**Status:** PERIODIC (re-emerges with each new alternative medicine trend)

#### Example: Gerson Therapy Misinformation
- **False Claim:** "Gerson therapy cured this advanced cancer patient" (with before/after X-rays)
- **Reality:** 
  - "Before" image = stage III lung cancer (confirmed)
  - "After" image = different patient entirely OR same patient with actual chemo
- **Fact-Checking Source:** Cancer Research UK, NIH cancer.gov
- **Lesson for Collection:** Same image anatomy reused with false attribution

### 2.3 Case Study 3: Reproductive Health Misinformation

**Status:** ENDEMIC (affects maternal health decisions)

#### Example: Ultrasound Misinterpretation Claims
- **False Claim:** "This ultrasound shows fetal abnormality from vaccine exposure"
- **Reality:** Normal anatomical variant or measurement error in interpretation
- **Vulnerable Population:** Pregnant women; particularly effective because stakes are high
- **Platform:** WhatsApp groups, Facebook pages
- **Geographic Focus:** Africa, South Asia (where ultrasound access is limited)

## PART 7: ETHICS & PRIVACY PROTOCOLS

### 7.1 Image Usage Compliance

**For each dataset/image collection:**

- [ ] **Verify License Compliance**
  - CC-BY: Can use with attribution
  - CC-BY-NC: Cannot use for commercial purposes (your work is likely non-commercial/academic)
  - Proprietary: Requires permission
  - Unlicensed: Avoid or seek permission

- [ ] **Respect Patient Privacy**
  - Dataset images should be de-identified (check metadata)
  - If DICOM files: Strip DICOM tags containing patient info
  - If collecting from WhatsApp/social media: Further anonymize; do not identify patients

- [ ] **IRB Consideration**
  - If you're publishing this work (likely), check if IRB approval needed
  - Using publicly available, de-identified data: Usually exempt
  - Using WhatsApp group images (even de-identified): Likely needs ethics review
  - **Action:** Consult with UBC Research Ethics Board before Phase 2 data collection

### 7.2 Misinformation Example Collection

**If collecting examples from WhatsApp groups / Facebook groups:**

1. **Informed Consent**
   - Get explicit consent from group members
   - Document in ethics protocol

2. **Minimize Identifiability**
   - Redact usernames, profile pictures, user IDs
   - Store de-identified version only
   - Keep consent log separately

3. **Purpose Limitation**
   - Use data only for stated research purpose
   - Don't share raw data with third parties
